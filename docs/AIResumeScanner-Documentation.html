AI Resume Scanner — Documentation

Version: 2025-08-17

Overview

The AI Resume Scanner analyzes and ranks resumes against a job description using NLP, providing interactive charts and an AI shortlisting assistant within a modern dark-themed Streamlit UI.

Features

  • Upload multiple resumes (PDF/DOCX), parse text, extract skills/entities
  • TF‑IDF similarity with skill matching and ranking
  • Plotly charts (dark theme) for matches and skills
  • Assistant with quick prompts, context toggles, and export
  • JSON persistence, no external database required

Setup (Windows PowerShell)

  1. Create and activate a venv
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
  1. Install dependencies
pip install -r requirements.txt
  1. Run the app
.\.venv\Scripts\python -m streamlit run .\app.py --server.port 8502

Open http://localhost:8502

Usage

  1. Paste a job description
  2. Upload one or more resumes (PDF/DOCX)
  3. Adjust filters (minimum match, top N)
  4. Review results and charts
  5. Use the assistant to compare, list skills, or summarize
  6. Browse Analysis History

Configuration

  • .streamlit/config.toml: theme and server (port/address)
  • Data files in data/, stored as JSON
  • No API keys required; assistant uses public endpoint with fallback

Modules

  • app.py — UI orchestration, dark theme, assistant panel
  • ranking_system.py — TF‑IDF + cosine similarity, ranking
  • resume_parser.py — PDF/DOCX text extraction
  • nlp_processor.py — preprocessing, skills/entities
  • utils.py — AI call + helpers
  • database.py — JSON persistence helpers

Data Formats

See docs/DATA_FORMATS.md for full JSON examples.

Troubleshooting

  • Activate venv and reinstall if modules are missing
  • Change port if 8502 is busy
  • Use text-based PDFs (OCR if needed)
  • Assistant will fall back to rule-based if network is unavailable

Security & Privacy

  • Local processing by default; data stored under data/
  • Avoid sensitive PII; clear data regularly
  • Public inference call can be disabled in utils.py
